# 双曲检索修复版本 · 运行说明

一份自包含的操作手册。包含目录组织、使用方式、以及**建议的运行顺序**（从最快验证到最深入改动）。

---

## 一、组织方式

### 1.1 顶层文件

```
algorithms/
├── 运行说明.md            ← 本文件（操作手册）
├── README.md              ← 版本索引 + 兼容矩阵 + 修改依据
├── EXPERIMENTS.md         ← 10 个实验的详细命令
├── apply.py               ← 统一应用脚本（自动 backup）
├── revert.py              ← 还原所有改动
├── diagnose.py            ← 单次检索诊断工具
│
└── v1~v9 子目录           ← 每个版本一个目录
```

### 1.2 每个版本子目录的结构

```
v<N>_<名字>/
├── README.md              ← 问题、修改、如何验证、需不需要重训
├── manifest.json          ← 文件映射表（apply.py 读取）
└── <修改后的源文件>        ← 每个版本改 1~3 个文件
```

`manifest.json` 示例（v1）：
```json
{
  "name": "v1_softplus_fix",
  "description": "反解 softplus...",
  "files": {
    "hyperbolic_projector.py": "model/hyperbolic_utils/hyperbolic_projector.py"
  },
  "post_apply_instructions": "..."
}
```

键=版本目录里的文件名；值=相对于项目根的目标路径。`apply.py` 读这个表做替换。

### 1.3 九个版本一览

| 版本 | 一句话 | 类型 | 需重训 |
|---|---|---|---|
| v1_softplus_fix | 让 `initial_curvature=0.1` 真正等于 0.1，而非 `softplus(0.1)≈0.74` | 训练 | 是 |
| v2_mask_fix | `parent_child_mask` 修正假负例（多父共子时漏标的真父子对）| 训练 | 是（主要 phase 2）|
| v3_mixed_training | 顺序训练改成按权重随机交错，防灾难性遗忘 | 训练 | 是 |
| v4_query_prefix | 给 query 加前缀，对齐节点 `level_embedding` 分布（探索性）| 推理 | 否 |
| v5_inference_sim | 推理 sim 公式从 `cos(α_Q-α_D)` 改为 `exp(-α_D/τ)` | 推理 | 否 |
| v6_parent_gate | 父权重加 "query 在父锥内" 门控，丢掉无关父 | 推理 | 否 |
| v7_domain_geodesic | DOMAIN 层改走测地线（止血方案）| 推理 | 否 |
| v8_depth_range | 去 projector 硬投影，改软 MSE；层内径向自由度恢复 | 训练 | 是 |
| v9_full_chain | 加入跨层级对（DOMAIN-KEYWORD、CATEGORY-DIALOGUE）训练 | 训练 | 是 |

## 二、使用方式

### 2.1 基本命令三件套

```bash
# 应用某版本（会自动 backup 被替换的文件）
python algorithms/apply.py <版本名>

# 还原所有改动（扫所有 *.v_original 备份恢复）
python algorithms/revert.py

# 还原单个文件
python algorithms/revert.py model/hyperbolic_utils/hyperbolic_projector.py
```

### 2.2 典型工作流

```bash
# 1. 保证干净起点
python algorithms/revert.py

# 2. 应用一个版本
python algorithms/apply.py v7_domain_geodesic

# 3a. 纯推理版本：直接跑检索对比
python algorithms/diagnose.py "When did Caroline go to the LGBTQ support group?" \
    --retriever_type hyperbolic_angular_geodesic_hybrid \
    --checkpoint /path/to/hyperbolic_projector_final.pt \
    --persist_dir /path/to/hierarchical_memory_locomo

# 3b. 需重训版本：先训练再跑检索
python -m model.hyperbolic_utils.train \
    --vector_store_path /path/to/store \
    --output_dir ./checkpoints_v7 \
    [其他训练参数]
python algorithms/diagnose.py "..." --checkpoint ./checkpoints_v7/hyperbolic_projector_final.pt ...

# 4. 做完这一版实验，还原，进入下一版
python algorithms/revert.py
python algorithms/apply.py <下一个版本>
```

### 2.3 备份机制说明

- `apply.py` 把目标文件复制到同目录下 `<name>.v_original` 作为 backup。
- 如果 backup 已存在（上一次 apply 留下的），**不会**被覆盖——永远保留"最原始"的版本。
- `revert.py` 读 `*.v_original` 还原后**删除** backup。
- 所以 "apply v5 → apply v6" 会让 v5 的改动被 v6 覆盖；想组合多个同文件版本，需手动合并（见下）。

### 2.4 多个版本组合

**情况 A：改不同文件**（例：v1+v2+v3 都只改各自文件，**直接叠加**）
```bash
python algorithms/apply.py v1_softplus_fix
python algorithms/apply.py v2_mask_fix
python algorithms/apply.py v3_mixed_training
# 三份 backup 并存，revert.py 会一次还原全部
```

**情况 B：改同一文件**（例：v5+v6+v7 都改 `hyperbolic_retriver.py`）

直接串行 apply 会丢失前一版的改动。**需要手动合并**。

推荐步骤：
1. `python algorithms/apply.py v5_inference_sim` — 把 v5 的版本装上
2. 打开 `algorithms/v6_parent_gate/hyperbolic_retriver.py`，找到标了 `v6_parent_gate` 注释的改动段，把它们**手动复制**到被 v5 改过的 `model/retrievers/hyperbolic_retriver.py`
3. 同理合并 v7 的改动
4. 此时 `model/retrievers/hyperbolic_retriver.py` 已含 v5+v6+v7 所有改动，backup 仍是 v5 装上前的原版
5. 跑实验
6. 完事 `python algorithms/revert.py` 还原到原版

各个版本的改动都在同一文件的**不同函数**里，合并冲突极低：
- v5 改：`MultiParentAngularHyperbolicRetriever.__init__` 和 `_weighted_cosine_angular_opposite_score`
- v6 改：`MultiParentAngularHyperbolicRetriever.__init__`（少量）和 `_parent_aggregation_weights`
- v7 改：`HybridHyperbolicRetriever._similarity` 和 `set_hybrid_scoring_boundary`、默认 boundary

v5 和 v6 只在 `__init__` 的参数列表有小冲突，合并时把两边的新参数都加上即可。

### 2.5 诊断工具 `diagnose.py`

用法：
```bash
python algorithms/diagnose.py "<查询文本>" \
    --retriever_type <cosine|hyperbolic_geodesic|hyperbolic_angular|hyperbolic_angular_geodesic_hybrid> \
    --checkpoint <projector_checkpoint_path> \
    --persist_dir <vector_store_directory> \
    [--top_k 10] \
    [--query_prefix "QUERY: "]    # v4 启用后有效
```

输出：每层候选数、top-k 得分范围（**dynamic range** 是关键指标）、每个 hit 的父节点和内容。

判断改进方向：比较两次运行的 `dynamic range`。**DOMAIN 层动态范围从 0.008 级别提升到 >0.03 是成功的关键指标**。

## 三、建议的运行顺序

按"**验证快 → 花费大**"的顺序。每一步都可以先做、先跑出结论，再决定要不要继续。

### 阶段 0：建立基线（必做）

**目的**：记录未改动状态下的各项数值，作为对照。

```bash
python algorithms/revert.py   # 保证干净

# 对两个已知答案的查询各跑一次，每种 retriever_type 都跑
for rt in cosine hyperbolic_geodesic hyperbolic_angular hyperbolic_angular_geodesic_hybrid; do
    echo "===== retriever=$rt ====="
    python algorithms/diagnose.py "When did Caroline go to the LGBTQ support group?" \
        --retriever_type $rt --checkpoint ... --persist_dir ... > baseline_${rt}_q1.txt
    python algorithms/diagnose.py "What fields would Caroline be likely to pursue in her education?" \
        --retriever_type $rt --checkpoint ... --persist_dir ... > baseline_${rt}_q2.txt
done
```

**关注指标**：
- 各层 dynamic range
- 正确祖先（如 `domain_2`、`domain_8`）是否进入 top-10
- 最终 DIALOGUE 层是否命中真证据（dialogue_4 / dialogue_8 / dialogue_10）

基线记好后，后面每一步实验都和它比。

### 阶段 1：纯推理修复（30 分钟内能跑完，无需重训）

**目的**：不碰 projector，只改检索公式。最快排除"推理公式错位"这一假设。

#### 1.1 v7 — DOMAIN 层走测地线（最单点）

```bash
python algorithms/apply.py v7_domain_geodesic
python algorithms/diagnose.py "When did Caroline..." \
    --retriever_type hyperbolic_angular_geodesic_hybrid \
    --checkpoint ... --persist_dir ...
# 记录结果
python algorithms/revert.py
```

**看什么**：Q1 的 DOMAIN 层 top-10 里是否出现 `domain_2`。若是，证明 DOMAIN 的 O 内角路径确实是一个显著瓶颈；若不是，说明问题更深（projector 本身训得不好）。

#### 1.2 v5 — 推理 sim 公式对齐训练目标

```bash
python algorithms/apply.py v5_inference_sim
python algorithms/diagnose.py "When did Caroline..." \
    --retriever_type hyperbolic_angular --checkpoint ... --persist_dir ...
python algorithms/revert.py
```

**看什么**：CATEGORY 层 top-10 得分是否从 `0.98~0.999` 压缩带扩展到 `0.x~0.y` 的有分辨力分布。

#### 1.3 v6 — 父权重锥内门控

```bash
python algorithms/apply.py v6_parent_gate
python algorithms/diagnose.py "..." --retriever_type hyperbolic_angular ...
python algorithms/revert.py
```

**看什么**：对有多父的候选（`keyword_13 support` 有 30+ 父），打印权重分布，看是否集中到 1-2 个相关父。

#### 1.4 三件套组合：v5+v6+v7（手动合并）

按 §2.4 情况 B 的流程手动合并。期望得到**推理侧能做到的最好效果**。若仍显著弱于 cosine 基线，说明 projector 本身需要重训（进入阶段 2+）。

#### 1.5 v4 — query 前缀扫描（可选）

```bash
python algorithms/apply.py v4_query_prefix
for prefix in "" "QUERY: " "DOMAIN: " "CATEGORY: "; do
    python algorithms/diagnose.py "..." --query_prefix "$prefix" ...
done
python algorithms/revert.py
```

**看什么**：是否某个前缀下 DOMAIN dynamic range 显著最大。若所有前缀都差不多，说明前缀不是关键变量（和你的直觉一致），可以搁置。

### 阶段 2：小成本训练修复（每项重训 1-2 小时）

**目的**：不动 projector 架构或损失设计，只修训练流程里的明显 bug。

#### 2.1 v1 — softplus 反解

```bash
python algorithms/apply.py v1_softplus_fix
python -m model.hyperbolic_utils.train \
    --vector_store_path ... --output_dir ./checkpoints_v1 \
    --initial_curvature 0.1
# 训练日志第一步应该打印 curvature: 0.1000（不是 0.7444）

python algorithms/diagnose.py "..." --checkpoint ./checkpoints_v1/hyperbolic_projector_final.pt ...
python algorithms/revert.py
```

**看什么**：训练后各层节点的平均深度是否从 `0.116 / 0.348 / 0.58 / 0.81` 变成 `0.316 / 0.949 / 1.58 / 2.21`（前者是 R=1.16 下的，后者是 R=3.16 下的）。检索质量变化。

#### 2.2 v2 — mask 假负例修复

```bash
python algorithms/apply.py v2_mask_fix
# 重训 phase 2 为主（多父共子最密集的层）
python -m model.hyperbolic_utils.train \
    --vector_store_path ... --output_dir ./checkpoints_v2 \
    --level_pair_index 2
# 后面的 phase 3 要不要重训看情况
python algorithms/diagnose.py "..." --checkpoint ./checkpoints_v2/... ...
python algorithms/revert.py
```

**看什么**：对多父候选（`keyword_13 support`）的检索一致性——所有真父对它的外角是否都变小。

### 阶段 3：训练策略重构（完整重训 3-6 小时）

#### 3.1 v3 — 混合批次训练

```bash
python algorithms/apply.py v3_mixed_training
python -m model.hyperbolic_utils.train \
    --mixed_training \
    --vector_store_path ... --output_dir ./checkpoints_v3 \
    --mixed_total_iterations 12500
# 训练日志应该每 100 步打印层级对采样比例，接近 {0.04, 0.32, 0.64}

python algorithms/diagnose.py "..." --checkpoint ./checkpoints_v3/... ...
python algorithms/revert.py
```

**看什么**：DOMAIN 层的 top-10 dynamic range 是否从 0.008 级别扩大到 0.05+。这是"灾难性遗忘"假说最直接的验证。

#### 3.2 v1 + v2 + v3 组合

三者改不同文件，可直接叠加。推荐同时开启做一个"训练侧最佳配置"基线：

```bash
python algorithms/apply.py v1_softplus_fix
python algorithms/apply.py v2_mask_fix
python algorithms/apply.py v3_mixed_training

python -m model.hyperbolic_utils.train \
    --mixed_training --initial_curvature 0.1 \
    --vector_store_path ... --output_dir ./checkpoints_v1v2v3
python algorithms/diagnose.py "..." --checkpoint ./checkpoints_v1v2v3/... ...
python algorithms/revert.py
```

### 阶段 4：架构级改造（完整重训，观察变化最大）

#### 4.1 v8 — 软化 projector（去硬投影）

```bash
python algorithms/apply.py v8_depth_range
python -m model.hyperbolic_utils.train \
    --vector_store_path ... --output_dir ./checkpoints_v8
# 验证层内径向分布：同层 ||z|| 的 std 应该从 0.0001 提升到 0.02+
python algorithms/revert.py
```

**看什么**：
- 同层 `||z_H||` 的标准差（目标 > 0.02）
- 层间均值顺序是否保持（DOMAIN < CATEGORY < KEYWORD < DIALOGUE）
- 检索质量

#### 4.2 v9 — 全链条训练

```bash
python algorithms/apply.py v9_full_chain
python -m model.hyperbolic_utils.train \
    --chain_training --chain_skip_weight 0.4 \
    --vector_store_path ... --output_dir ./checkpoints_v9 \
    --mixed_total_iterations 15000
# 日志应该显示 5 个 level_pair（3 相邻 + 2 跨层）的采样计数

python algorithms/diagnose.py "..." --checkpoint ./checkpoints_v9/... ...
python algorithms/revert.py
```

**看什么**：
- 祖-孙外角（比如 DOMAIN → KEYWORD）分布是否更集中
- 完整 DOMAIN→CATEGORY→KEYWORD→DIALOGUE 链的召回

### 阶段 5：综合最佳配置

根据前面每一版的观察，选出有效的组合，最终组合跑一次：

```bash
# 示例：若 v1、v2、v3、v9 都有效果
python algorithms/apply.py v1_softplus_fix
python algorithms/apply.py v2_mask_fix
# v3 和 v9 都改 hierarchical_dataset.py 和 train.py，需手动合并
# v9 自带 v3 的功能，可直接 apply v9 覆盖 v3
python algorithms/apply.py v9_full_chain

python -m model.hyperbolic_utils.train \
    --chain_training --initial_curvature 0.1 \
    --vector_store_path ... --output_dir ./checkpoints_best

# 推理侧也挂上 v5/v6/v7（手动合并）
# 跑综合诊断
python algorithms/diagnose.py "..." --checkpoint ./checkpoints_best/... ...
```

## 五、完整流程模板（一键跑所有阶段）

把下面保存为 `scripts/run_all_experiments.sh` 之类的脚本，按需注释：

```bash
#!/usr/bin/env bash
set -e

CKPT_BASE=/path/to/original/hyperbolic_projector_final.pt
STORE_DIR=/path/to/hierarchical_memory_locomo
Q1="When did Caroline go to the LGBTQ support group?"
Q2="What fields would Caroline be likely to pursue in her education?"

# 阶段 0
python algorithms/revert.py
python algorithms/diagnose.py "$Q1" --retriever_type hyperbolic_angular \
    --checkpoint $CKPT_BASE --persist_dir $STORE_DIR > logs/q1_baseline.txt

# 阶段 1：逐个推理版本
for v in v7_domain_geodesic v5_inference_sim v6_parent_gate; do
    python algorithms/revert.py
    python algorithms/apply.py $v
    python algorithms/diagnose.py "$Q1" --retriever_type hyperbolic_angular \
        --checkpoint $CKPT_BASE --persist_dir $STORE_DIR > logs/q1_$v.txt
done

# 阶段 2+：需重训的版本（自行按需打开）
# python algorithms/revert.py
# python algorithms/apply.py v1_softplus_fix
# python -m model.hyperbolic_utils.train --vector_store_path $STORE_DIR --output_dir ./checkpoints_v1
# python algorithms/diagnose.py "$Q1" --checkpoint ./checkpoints_v1/hyperbolic_projector_final.pt ... > logs/q1_v1.txt

python algorithms/revert.py
echo "Done. 对比 logs/ 下的输出。"
```

## 六、常见问题

**Q：apply.py 提示"版本目录不存在"？**
A：检查拼写。版本名要完整，包括 `v1_softplus_fix` 这种格式，不是只写 `v1`。

**Q：apply 后训练报错 `TypeError: too many values to unpack`？**
A：你应用的是 v8，但 `hyperbolic_retriver.py` 还是老版本（v8 改了 projector 返回值）。v8 的 train.py 已经同步处理；如果你走 `llm_inference` 推理，也要确保 retriever 正确读 projector 输出。v8 设计上让 projector 仍返回 2-tuple（存 `_last_target_norm` 为 attribute），所以推理端**不需要**改。若仍报错，检查你是否有旧的 pyc 缓存：`find . -name "*.pyc" -delete`。

**Q：想同时跑两个版本做 A/B 测试？**
A：Apply 两次会丢 backup。推荐用 git worktree 或 git branch 做隔离：
```bash
git checkout -b experiment-v7
python algorithms/apply.py v7_domain_geodesic
# 跑实验
git stash; git checkout -b experiment-v5
python algorithms/revert.py  # 这里 revert 只影响当前 worktree
python algorithms/apply.py v5_inference_sim
# 跑实验
```

**Q：backup 文件能删吗？**
A：能。`revert.py` 会读 `*.v_original` 还原并删除。若想强制丢弃某版本，手动 `rm <file>.v_original` 即可。但一旦删了 backup，就无法自动还原到原版了。

**Q：每个版本的 README 里的"期望效果"是实测还是预测？**
A：预测。基于 `结果(3).md` 的现有观察做的推理。实际效果需要你跑出来验证。若某版本跑完反而变差，请把现象贴出来，可能是我对某个机制的理解偏了。
